# What's New

## Unicode Filename Support

ECM-R no longer crashes when MP3 files contain Cyrillic, Korean, Japanese, or other non-ASCII characters in their filenames. The game will now enumerate, display, and play these files correctly.

### Background

The game engine uses ANSI (CP-ACP) internally, which cannot represent most non-Western characters. ECM-R now stores all file paths as UTF-8 internally and converts only at the boundaries where required.

### Playlist Display

- The ECM-R in-game overlay now renders Unicode filenames correctly (Cyrillic, Korean, Japanese, etc.) in the playlist menu.
- Fixed a bug where filenames with multiple ` - ` separators (e.g. `Artist - Title - Subtitle`) showed as `N/A` in the notification.

### In-Game Notification (Chyron)

- The chyron shows the parsed title and artist for every track.
- Characters that cannot be represented in the game's ANSI code page display as `?`. This is a game engine limitation and affects all mods that interact with the stock notification system.

### INI Persistence

- The `[trax]` section now stores filenames as UTF-8, which is backwards compatible with existing INI files.

## Bugs Fixed

- Fixed a crash on startup when any audio file has non-ASCII characters in its name.
- Fixed the file extension filter that was incorrectly accepting all file types regardless of the configured extension list.
- Fixed the title parser that corrupted some track titles by trimming the first two characters.
