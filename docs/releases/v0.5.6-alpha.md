# What's New

## New Features

### Manual Pause and Resume Control

- Added **Pause/Resume** playback control to the ECM-R overlay.
- Added a new configurable hotkey, `pause_track`, with default value `F8`.
- Manual pause is now user-controlled and remains active until the user explicitly resumes playback.
- The main overlay bar now shows the current playback state at a glance:
  - `Paused`
  - `Playing`

### Overlay and Support Improvements

- Added an **About** menu to the overlay.
- The About menu keeps ECM-R fork attribution visible in-game.
- Added quick links to the repository and issue tracker from the overlay.

## Playback Behavior Changes

### Pause State Across Context Changes

- Manual pause now persists across playlist context changes such as frontend and in-game transitions.
- If the user pauses playback in one context and resumes in another, ECM-R evaluates the paused track against the current playlist context.

**Resume behavior:**
- If the paused track is still valid for the current context, ECM-R resumes that same track.
- If the paused track is no longer valid for the current context, ECM-R continues with the next valid track for the active context.
- When a new track must be selected after resume, the result respects the current **shuffle** and **repeat** settings.

### In-Game Chyron Handling

- Pausing playback now hides the in-game chyron notification instead of leaving an empty notification visible.
- Resuming playback restores the chyron for the active track.
- Previous and next track changes continue to update the chyron as expected.

## Configuration Changes

### Updated `[keys]` Section

The generated `ecm-r.x86.ini` now includes:

```ini
[keys]
toggle_overlay = F11
pause_track = F8
previous_track = F9
skip_track = F10
```

**New setting:**
- `pause_track`: Key used to pause or resume ECM-R playback manually. Default: `F8`

### Backward Compatibility

- Existing configurations automatically gain the `pause_track` entry on next launch when needed.
- Existing shuffle, repeat, volume, and playlist settings remain unchanged.
- Manual pause state is runtime-only and is not persisted across game restarts.

## Technical Changes

### Pause State Management

- Separated manual pause behavior from automatic gameplay-driven pause handling.
- Resume logic now validates the paused track against the current playlist context before continuing playback.
- If the paused track is invalid after a context change, ECM-R falls back to the normal playlist flow for the current context.

### Input and Overlay Integration

- Extended the input system to support the new `pause_track` binding.
- Added Pause/Resume handling to both keyboard input and the overlay Actions menu.
- Added a visible playback status label in the main overlay bar.

### Frontend Integration

- Updated chyron handling so pause removes the active frontend/in-game music notification cleanly.
- Resume restores the current track notification without requiring a track skip.

## Documentation Updates

- Updated `README.md` to document the new Pause/Resume feature, default `F8` binding, overlay playback state, and About menu.
- Updated `CONFIGURATION.MD` to document the `pause_track` setting and manual pause behavior across context changes.
- Updated `CHANGELOG.md` for `v0.5.6-alpha`.

## How to Use

### Pause or Resume from the Hotkey

1. Press `F8` to pause ECM-R playback.
2. Press `F8` again to resume playback.

### Pause or Resume from the Overlay

1. Press `F11` (default) to open the ECM-R overlay.
2. Go to **Actions**.
3. Click **Pause** to pause playback.
4. Click **Resume** to continue playback.

### Change the Pause Hotkey

1. Open `ecm-r.x86.ini`.
2. Edit `pause_track` under `[keys]`.
3. Restart the game.

## Behavior Examples

### Resume in the Same Context

```text
Current context: IG
Current track: Track A (IG)
Action: Pause -> Resume
Result: Track A resumes
```

### Resume After a Context Change with a Valid Shared Track

```text
Current context: IG
Current track: Track A (ALL)
Action: Pause -> switch to FE -> Resume
Result: Track A resumes because it is still valid in FE
```

### Resume After a Context Change with an Invalid Track

```text
Current context: IG
Current track: Track B (IG)
Action: Pause -> switch to FE -> Resume
Result: Track B does not resume; ECM-R continues with the next valid FE/ALL track
```

### Resume After Changing Shuffle While Paused

```text
Current context: IG
Shuffle: On
Current track: Track C (IG)
Action: Pause -> switch to FE -> disable shuffle -> Resume
Result: If Track C is invalid in FE, ECM-R continues using the current FE playlist flow without shuffle
```

## Notes

- Manual pause does not disable or overwrite shuffle and repeat; it defers playback until the user resumes.
- If resume must pick a different track because of a context change, the result respects the playback mode active at that moment.
- The gameplay-facing pause state only applies to ECM-R playback and does not alter the game's own pause systems.
- The in-game notification is hidden while playback is manually paused and restored on resume.
