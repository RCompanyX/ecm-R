# What's New

## Startup Release Notice

### One-Time GitHub Check

- ECM-R now resolves release availability once during overlay startup instead of retrying during per-frame UI updates.
- The lookup runs asynchronously so the overlay stays responsive while the request is in flight.
- WinHTTP timeouts are kept short so unreachable networks fail fast and gameplay continues without a startup popup.

### Stable vs Testing Status

- The startup notice now states whether the newer GitHub build is a **stable release** or a **testing pre-release**.
- The tooltip mirrors that status and shows the detected tag.
- The notice stays hidden when the current build is already up to date or the release check cannot complete.

## Release Discovery Policy

### Latest Non-Draft Release

- Release discovery now reads the GitHub releases list instead of `/releases/latest`.
- ECM-R explicitly follows the latest non-draft release so published testing builds remain visible in the overlay.
- Version comparison now evaluates numeric components and prerelease suffixes before deciding that a newer release exists.

## Documentation Updates

- Updated `README.md` to document the startup release notice and its stable/testing wording.
- Updated `CHANGELOG.md` for `v0.5.10-alpha`.

## Validation

- Verified the touched source and documentation files for editor-reported errors.
- Queried the live GitHub releases list to confirm the current repository release flow exposes the `draft` and `prerelease` flags used by the new discovery policy.

## Notes

- Current release policy is **latest non-draft**, not **latest stable only**.
- If the release check fails, ECM-R shows no release notice and continues without a popup.
