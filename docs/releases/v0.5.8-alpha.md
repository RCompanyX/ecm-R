# What's New

## Hotkey Rebinding and New Toggle Shortcuts

### Runtime Hotkey Configuration

- Added a dedicated **Hotkeys** menu to the ECM-R overlay.
- The overlay can now rebind all existing ECM-R hotkeys at runtime:
  - `toggle_overlay`
  - `pause_track`
  - `previous_track`
  - `skip_track`
- Added two new optional keyboard toggles:
  - `toggle_shuffle`
  - `toggle_repeat`

### New Default Key Behavior

- The new `toggle_shuffle` and `toggle_repeat` entries are created as `None` by default.
- This avoids introducing new in-game key conflicts until the user assigns those actions manually.
- Existing configurations automatically gain those two new `[keys]` entries on update.

## Overlay Rebinding Behavior

### Safe Capture Flow

- While the overlay is waiting for a new key, ECM-R suspends all ECM-R hotkey execution.
- Pressing the candidate key during capture no longer triggers playback, overlay, shuffle, or repeat actions.
- Capture can be canceled explicitly from the UI without consuming a keyboard key.

### Validation and Safeguards

- Duplicate key assignments are rejected directly in the overlay.
- Resetting a binding back to its default also respects the duplicate-assignment rule.
- `toggle_overlay` cannot be cleared from the UI, which avoids stranding the user without a way to reopen the overlay.
- If a hotkey save fails, ECM-R restores the previous runtime binding instead of leaving runtime and INI state out of sync.

## Shared Shuffle and Repeat Behavior

### Keyboard and Overlay Stay Aligned

- Shuffle and repeat now use shared runtime helpers.
- The new shuffle hotkey performs the same work as the overlay checkbox:
  - toggles `shuffle_enabled`
  - rebuilds the playlist order
  - resets the current playlist index
- The new repeat hotkey uses the same toggle path and persists the updated setting immediately.

## Configuration Changes

### Updated `[keys]` Section

The generated `ecm-r.x86.ini` now includes:

```ini
[keys]
toggle_overlay = F11
pause_track = F8
previous_track = F9
skip_track = F10
toggle_shuffle = None
toggle_repeat = None
```

### Accepted Unassigned Syntax

- `None` and `Unbound` are both accepted as explicit disabled values.
- Unsupported or duplicate key entries are normalized back to safe defaults when the configuration is loaded.
