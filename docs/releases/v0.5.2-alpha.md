# What's New

## New Features

### Shuffle and Repeat Playback Modes

- Added configurable `shuffle_enabled` and `repeat_enabled` playlist options.
- Added overlay controls to toggle **Shuffle** and **Repeat** at runtime.
- Shuffle can now be disabled for sequential playback order.
- Repeat can now be disabled so playback stops after the last valid track in the current context.

### Configuration Changes

#### Updated `[config]` Section

The generated `ecm-r.x86.ini` now includes:

```ini
[config]
shuffle_enabled = true
repeat_enabled = true
```

**Accepted values:**
- `true`, `false` (boolean strings)
- `1`, `0` (numeric)
- `yes`, `no`, `on`, `off` (aliases)

**Defaults:**
- `shuffle_enabled = true`
- `repeat_enabled = true`

### Backward Compatibility

- Existing configurations automatically gain the new playback options on next launch when needed.
- Overlay changes are saved back to `ecm-r.x86.ini` automatically.
- Previous ECM-R playback features remain unchanged.

## Technical Changes

### Playlist Behavior

- Enhanced playlist ordering so valid tracks can play shuffled or sequentially depending on configuration.
- Added repeat-aware end-of-playlist handling.
- Playlist state now persists the new playback options through the configuration file.

### Overlay Integration

- Added **Shuffle** and **Repeat** checkboxes to the Actions menu.
- Added playback mode status text for current random/sequential and repeat state.

## Documentation Updates

- Updated `CHANGELOG.md` for `v0.5.2-alpha`.
- Updated `README.md` with the new playback mode settings and overlay controls.
- Updated `BUILDING.md` to document the generated configuration entries.

## How to Use

1. **Default behavior:** No action required. Shuffle and repeat are enabled automatically.

2. **To change playback mode from the overlay:**
   - Open the ECM-R overlay.
   - Go to **Actions**.
   - Toggle **Shuffle** or **Repeat** as needed.

3. **To change playback mode from the configuration file:**
   - Open `ecm-r.x86.ini`.
   - Edit `shuffle_enabled` and `repeat_enabled`.
   - Restart the game.

## Notes

- Shuffle and repeat are enabled by default to preserve the expected ECM-R playback experience.
- Sequential mode respects the discovered valid track order for the active context.
- Disabling repeat stops playback after the final valid track instead of restarting the list.
