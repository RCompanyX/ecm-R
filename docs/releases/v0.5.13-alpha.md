# What's New

## Song Transition Smoothing

ECM-R has improved music loading across various scenarios where brief freezes occurred when switching tracks. The unnecessary `BASS_STREAM_PRESCAN` flag was performing a blocking file pre-scan during stream creation, which stalled the game's main loop. Removing this flag resolves the most common causes of stutter.

### Background

The `BASS_STREAM_PRESCAN` flag tells BASS to scan the entire audio file before returning the stream handle. For large or remotely hosted files this can be useful, but for local music files the pre-scan added a noticeable delay without user-facing benefit. The stream is now opened with default flags and plays immediately.

### Ongoing Work

Optimization is still being explored. In isolated cases, some lag may still occur depending on file size, disk speed, and system load. Further refinements to the stream creation and playback pipeline are under investigation.

## Filename and Display Cleanup

ECM-R now parses filenames more reliably and displays them consistently across the overlay, playlist menu, and in-game notification.

### Robust Filename Parsing

- Filename parsing now splits on the first `-` character instead of the last, and trims whitespace from both sides. This correctly handles filenames with extra spaces around the separator, such as `04.    -   Song Test One.mp3` being displayed as `04. - Song Test One`.
- Artist and title values are now trimmed after parsing, so the overlay menu bar, playlist menu, and chyron notification show clean text without leading or trailing whitespace.

### Consistent Artist - Title Display

- The overlay menu bar previously showed only the song title; it now shows the artist alongside the title in `Artist - Title` format when available, matching the in-game chyron behavior.
- The playlist menu no longer shows raw filenames with extra whitespace but instead displays the cleaned `Artist - Title` format that users see everywhere else.
