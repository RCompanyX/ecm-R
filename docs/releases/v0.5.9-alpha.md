# What's New

## Better BASS Load Diagnostics

### Clearer Startup Failure Popup

- When ECM-R cannot load `bass.dll`, the startup popup now uses clearer wording.
- The popup keeps the localized Windows error text returned by the loader.
- The popup now also shows the exact path that ECM-R tried to load.
- The message points users back to the README for official `bass.dll` download and placement guidance.

## Documentation Updates

### Roadmap Refresh

- The README roadmap now lists **In-Game Movie Handling** as planned work.

### BASS Setup Guidance

- The README notes now explain that failed `bass.dll` loads show both the Windows error text and the attempted path.

## Technical Changes

### Stable Loader Error Capture

- `GetLastError()` is now captured once before the BASS loader resets its internal state.
- This keeps the reported Windows loader error stable in failure cases.

## Validation

- Built successfully in `Release | Win-x86`.
- Reviewed the updated release and README text against the current `bass_api.cpp` and `audio.cpp` failure paths.

## Notes

- This release does not change configuration keys or runtime file names.
- `bass.dll` must still be placed next to the ECM-R runtime files in the target `scripts` folder.
